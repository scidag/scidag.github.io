(1) 如何知道是哪个用户登录了？

连接服务器端后，得到一个 session 状态（匿名对话），返回给前端。
登录成功后，得到了登录成功的 session，并且给 session 设置一些值（比如用户信息等），返回给前端一个设置的 cookie 的 “命令”。 session ==> cookie
前端接受到后端命令后，设置 cookie，保存到浏览器内。
前端再次请求后端的时候（相同的域名），在请求头中带上 cookie 去请求。
后端拿到前端传来的 cookie，找到对应的 session。
后端从 session 中可以取出基于该 session 存储的变量（用户登录信息、登录名）
（2）Cookie 与 Session 区别
①Cookie

定义：Cookie 是一种存储用户本地计算机上的小数据片段。它由服务器发送，浏览器保存，并在后续请求中自动发送回服务器。Cookie 主要用于识别用户身份、存储用户偏好等。
特点：
存储在客户端
容量有限，大多数浏览器限制每个 Cookie 的大小不超过 4KB，且每个站点最多可以创建 20 个 Cookie
安全性较低，因为 Cookie 存储在用户的计算机上，可以被用户看到并修改，尽管可以通过设置 httpOnly 和 Secure 属性来提高安全性
发送 Cookie 回增加每次 HTTP 请求的数据量，可能影响页面加载速度
用途：
用户身份识别
存储用户偏好设置
跟踪用户行为
②session

定义：Session 是一种服务器端的技术，用于存储用户会话信息。服务器会为每个用户创建一个唯一的 Session ID，并通过 Cookie 或其他方式，如 URL 重写等，将 Session ID 发送给客户端。客户端在后续请求中会携带这个 session ID，服务器通过解析 Session ID 来获取对应的 Session 信息。
特点：
存储在服务器端
安全性较高，因为敏感信息不会直接存储在客户端
容量相对较大，因为受限于服务器的资源而非客户端浏览器限制
依赖于客户端的 Session ID 传递
用途：
存储用户会话信息，如登录状态、购物车内容等
管理用户登录状态，实现身份验证和授权
（3）cookie 和 session 的联系与区别
联系：
①session 通常依赖于 cookie 来传递 session ID，以维护用户会话的连续性。
②两者都可用于跟踪用户状态，但存储位置和数据量等方面有所不同

区别：
①存储位置：cookie 存储在客户端，session 存储在服务器端。
②安全性：session 相比于 cookie 安全性更高，因为敏感数据不直接存储在客户端
③容量：cookie 容量有限，session 的容量相对较大
④使用场景：cookie 更适合存储费敏感信息，如用户偏好；而 session 更适合存储敏感信息，如用户登录状态
